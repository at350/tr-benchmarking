STATUTE OF FRAUDS WORKED BENCHMARK EXAMPLES — INDEX + USAGE NOTES

Purpose:
This layer adds one clean worked benchmark example for each major Statute of Frauds provision. It sits after routing and doctrine-pack selection, but before variation. Its job is to give the GPT a concrete pattern for turning a doctrinal source into:
1. a neutral legal question,
2. a benchmark answer using the universal answer template, and
3. carry-forward variation notes that stay within the same doctrinal lane.

What this layer is for:
- showing one low-noise benchmark anchor per provision
- keeping answer headings stable across provisions
- making the controlling gate order concrete
- demonstrating how much detail belongs in a clean benchmark answer
- showing what can safely carry forward into Lane A and Lane B work later

What this layer is not for:
- it does not override the actual source materials or benchmark answer
- it does not supply jurisdiction-specific exceptions
- it does not authorize copying facts verbatim into new outputs
- it does not replace B01-B03

Recommended read/load order:
1. B00_SoF_Routing_Matrix_and_Universal_Template
2. B04_SoF_Pack_Map_and_Load_Order if used in your stack
3. the matching doctrine or provision pack by doctrine name
4. the matching worked benchmark example from B21-B26
5. B01_Variation_Modes_Guide_v2
6. B02_Menu_and_Output_Protocol
7. B03_Examples_and_RedFlags

Default use rule:
Load no more than one worked example unless the route is unstable. Match by doctrine name rather than by any particular numbering scheme.

Example map:
- Marriage provision -> B21_Marriage_Worked_Benchmark_Example
- Suretyship provision -> B22_Suretyship_Worked_Benchmark_Example
- One-year provision -> B23_One_Year_Worked_Benchmark_Example
- Land sale / interest in land provision -> B24_Land_Worked_Benchmark_Example
- UCC Article 2 / Section 2-201 -> B25_UCC_2_201_Worked_Benchmark_Example
- Executor / administrator personal-promise provision -> B26_Executor_Worked_Benchmark_Example

How to use a worked example:
1. Copy the structure, not the facts.
2. Keep the answer headings stable unless the assigned source requires a specific variant.
3. If the source materials conflict with the example, the source materials control.
4. Use the carry-forward variation notes only after the route and gate order are fixed.
5. By default, vary one fact family at a time.

Quality checks before reusing an example:
- Does the example match the same provision and the same main gate order?
- Do the facts stay on the same side of all threshold or classification lines?
- Are you importing an exception the actual source materials never used?
- Has a necessary fact been removed so that the task should be a Lane B ambiguity test rather than a Lane A safe variation?
- Are you silently changing party roles, liability direction, governing-law family, or writing status?

House rule:
These examples are intentionally clean and conservative. They are benchmark anchors, not creativity prompts. Their main value is to show what a stable benchmark packet looks like before controlled variation begins.
